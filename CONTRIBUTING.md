# Contributing to @forgesworn/shamir-core

## Getting Started

```bash
npm install
npm run build      # compile TypeScript → dist/
npm test           # run test suite (vitest)
npm run typecheck  # type-check without emitting
npm run clean      # remove dist/
```

Node.js >= 18 required. No other runtime dependencies.

## Project Structure

```
src/index.ts          — entire library (one file: GF(256), types, errors, split, reconstruct)
tests/index.test.ts   — full test suite (vitest)
dist/                 — compiled output (git-ignored, generated by npm run build)
```

## Code Conventions

- **British English** in all comments and docs — licence, serialise, behaviour, colour
- **Strict TypeScript** — `noUncheckedIndexedAccess` is enabled; handle all potential undefined accesses
- **Zero runtime dependencies** — do not introduce any; Web Crypto (`crypto.getRandomValues`) is the only external surface
- **Error hierarchy** — throw `ShamirValidationError` for bad inputs, `ShamirCryptoError` for internal arithmetic failures, never plain `Error`
- **Memory hygiene** — zero secret material (coefficients, intermediate buffers) in `finally` blocks using `Uint8Array.fill(0)`

## Commit Format

Commits must follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add optional share label field
fix: reject shares with empty data arrays
docs: clarify threshold semantics in README
refactor: extract Lagrange basis into helper
test: add cross-runtime reconstruction tests
```

Commit type drives semantic-release versioning: `feat` → minor bump, `fix` → patch bump, `feat!` or `BREAKING CHANGE:` footer → major bump.

## Branching and Release

- **Work on a branch.** Never commit directly to `main`.
- Semantic-release runs automatically on push to `main` — it publishes to npm, creates a GitHub release, and tags the commit. Do not manually bump the version in `package.json`.
- Open a pull request from your branch. All checks must pass before merging.

## Pull Request Checklist

- [ ] `npm run typecheck` passes with no errors
- [ ] `npm test` passes
- [ ] New behaviour is covered by tests
- [ ] Docs and comments use British English
- [ ] No new runtime dependencies added
- [ ] Commit messages follow the Conventional Commits format

## Security Reporting

Do not open a public issue for security vulnerabilities. Send a private report via GitHub's [security advisory](https://github.com/forgesworn/shamir-core/security/advisories/new) feature.

Please include a description of the vulnerability, reproduction steps, and your assessment of impact. We aim to acknowledge reports within 48 hours.
